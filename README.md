## Análise Comparativa

Q1 - Estruturas de Loop: 
Nessa questão eu achei Python a linguagem mais fácil de escrever, porque o código fica menor e mais simples de entender. Java já é mais verboso, tem mais coisa pra escrever. Haskell foi o mais difícil pra mim, porque é um estilo diferente (funcional). Swift achei bem organizado e fácil de ler.

Q2 - Seleção múltipla: 
Ruby foi o que eu achei mais fácil de entender, porque dá pra agrupar vários valores no mesmo caso. Em C funciona bem, mas fica mais repetitivo. Erlang eu achei mais complicado, porque precisa repetir mais os casos.

Q3 - Sem goto: 
Reescrever sem usar break deixou o código mais organizado. Tive que mudar a lógica do laço, usando condição no for/while. No geral, ficou mais fácil de entender depois de ajustar.

Q4 - For em múltiplas linguagens: 
JavaScript e C++ são bem parecidos. Python ficou diferente porque usei while. Go achei interessante por permitir várias variáveis no for. Kotlin também é bem limpo e moderno.

Opinião pessoal:
Eu achei Python e Ruby mais fáceis de trabalhar, principalmente por causa da simplicidade. Java e C são mais usados, mas exigem mais código. Haskell e Erlang foram os mais difíceis pra mim porque são menos intuitivos.
